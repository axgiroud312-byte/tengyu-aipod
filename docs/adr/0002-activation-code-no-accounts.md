# ADR-0002 — 微信登录 + 权益授权，兑换码仅作可选开通工具

**状态**：已采纳
**日期**：2026-05-23
**更新**：2026-05-30，废弃硬件机器码 / 设备指纹绑定，改为微信身份 + 权益模型。

## 背景

腾域 aipod 需要一种"付费 → 软件可用"的授权机制。常见做法：

1. **邮箱账号 + 订阅**：用户注册邮箱 + 密码，付费后开通订阅
2. **纯激活码**：管理员后台生成激活码，用户付款后填码即可
3. **微信登录 + 权益**：用户微信扫码登录，管理员可直接给该微信身份开通权益，必要时再用兑换码辅助开通
4. **硬件机器码 / 加密狗**：绑定 CPU / 主板 / 网卡或物理硬件

项目后续具备微信开放平台相关资质，因此可以把微信登录作为长期身份方案。支付能力 v1 仍然可以先不做自动扣费，继续由管理员后台人工开通权益。

## 候选方案与决策

### 选项 A：邮箱账号 + 订阅

需要：
- 邮箱注册、登录、找回密码
- 密码哈希、邮箱验证、风控
- Subscription 表 + 自助支付 / 续费机制

对当前用户群来说摩擦大，也不符合国内用户习惯。

### 选项 B：纯激活码

优点是实现简单；缺点是码本身承担身份，会带来丢码、转发、换机售后和客户归属不清的问题。若再叠加机器码绑定，会把授权问题变成低价值硬件指纹维护。

### 选项 C：微信登录 + 权益（**采纳**）

需要：
- WeChatUser 表（`unionid` 优先，必要时 fallback 到 `openid`）
- Entitlement 表（到期时间、模块权限、席位数、状态）
- ClientSession / InstallationSeat 表（登录态和席位占用）
- ActivationCode 表（仅作为兑换码或人工发码入口，不作为登录入口）
- 微信扫码登录回调 + 客户端 JWT

### 选项 D：硬件机器码 / 加密狗

不采用。硬件指纹容易误伤重装、换网卡、虚拟机、系统升级用户，也挡不住有动机的破解；加密狗成本高、交付麻烦。

## 决策

**采用微信登录 + 权益授权。兑换码只作为可选的权益开通工具，不再作为客户端登录入口；禁止使用 CPU / 主板 / 网卡等硬件机器码作为授权依据。**

核心链路：

1. 用户打开客户端，微信扫码登录。
2. 服务端根据 `unionid` / `openid` 找到或创建微信用户。
3. 用户若已有权益，直接签发客户端 JWT。
4. 用户若拿到兑换码，可在微信登录后输入兑换码，将权益绑定到微信用户或客户。
5. 客户端拉取 Skill、刷新登录态和执行关键操作时，服务端校验 JWT + 权益状态；Provider、模型清单和 Workflow 在本地管理。

## 理由

1. **身份更稳定**：人是主体，机器只是使用入口。换电脑、重装系统都不改变微信身份。

2. **用户体验更符合国内习惯**：扫码登录比邮箱注册、记密码、复制激活码更自然。

3. **售后更少**：不再处理机器码变化、硬件指纹误判、用户不会找机器码等问题。

4. **安全边界更真实**：防滥用靠服务端权益、席位、关键资源派发和联网校验，不靠伪安全的硬件哈希。

5. **销售仍然灵活**：v1 可以继续线下收款，管理员后台直接开通权益；推广、试用、渠道分发仍可用兑换码。

6. **后续扩展顺滑**：未来接自助支付、团队席位、员工管理、用户社区，都建立在微信身份之上。

## 影响

### 正面

- 首次启动变成微信扫码登录，减少手输码摩擦。
- 管理员后台按微信用户 / 客户 / 权益聚合，更容易看清真实客户关系。
- 兑换码泄露的影响降低：兑换码只能在微信登录后兑换，可禁用、可追踪批次。
- 不采集硬件机器码，隐私表述更干净。

### 负面

- 服务端需要接微信开放平台扫码登录。
- 客户端必须能打开登录页 / 扫码页并处理回调。
- 微信审核、域名、回调配置会成为上线前置条件。

### 缓解措施

- v1 支持管理员后台手动给微信用户或客户开通权益，不强依赖自助支付。
- 保留兑换码用于试用、推广和人工发码，不作为登录方式。
- 保留 7 天离线宽限，但拉取云端资源和关键操作必须联网校验。

## 席位问题

用户购买"主账号 + N 个子账号"实际等于一个或多个 **权益** 下的 `seat_count`。席位限制的是客户端安装或登录会话数量，不是硬件机器码。

后台必须支持：
- 查看某个微信用户 / 客户占用的席位
- 手动解绑旧席位
- 调整权益席位数
- 封禁某个微信用户或某个权益

## 替代决策的触发条件

考虑补充邮箱账号 / 手机号登录的信号：
- 海外用户明显增多，微信不可用
- 企业客户要求非微信员工账号体系
- 需要 SSO / 企业组织架构 / 审计能力
