日期：2026-07-02

## 背景

完整任务页当前在结果预览、逐项状态下方展示“最近完整任务”。该区域占用了首屏后半段空间，但对当前演示和当前运行帮助有限。用户希望取消历史任务卡片，把空间用于当前完整任务的实时成果展示，让客户一眼看到 Workbench 的处理效率和最终产出。

本设计只覆盖完整任务页的结果预览展示，不改变完整任务执行顺序、文件落盘规则、PS 套版输出结构或标题生成逻辑。

## 目标

1. 取消“最近完整任务”区域。
2. 扩大结果预览，让它成为当前完整任务的主要展示区域。
3. 默认展示当前任务的最终产物，而不是默认展示中间过程。
4. PS 套版开启时，用“成品英雄墙 + 文件夹陈列”的视觉形态展示套版成果。
5. 文生图、图生图、提取、抠图、检测等中间产物按实际开启情况展示，并默认收起。
6. 页脚统计按本次完整任务实际开启的步骤动态出现；未开启的步骤不显示、不占位。

## 非目标

1. 不实现新的完整任务编排引擎。
2. 不改变 `04-上架工作区` 的真实目录结构。
3. 不改变 PS 套版输出命名和多模板输出规则。
4. 不新增历史任务中心。
5. 不把上架接入完整任务。

## 展示原则

结果预览的主展示区永远展示“当前任务最终产物”：

| 本次任务开启情况 | 主展示默认内容 |
|---|---|
| 开启 PS 套版 | 套版后的 `模板批次 / 货号` 文件夹卡片 |
| 未开启 PS，开启侵权检测 | 检测通过的图片 |
| 未开启 PS 和检测，开启抠图 | 抠图结果 |
| 只跑文生图 | 文生图印花 |
| 只跑图生图 | 图生图印花 |
| 采集 + 提取后不跑后续 | 提取出的印花 |
| 已有印花来源后不跑后续 | 读取到的已有印花 |

如果多个阶段都开启，主展示取最后一个实际产生可继续产物的阶段。中间阶段仍保留在折叠区。

## 主视觉布局

采用 A+B 混合方案：

1. 左侧为成品英雄预览。
   - 展示当前选中的最终产物。
   - PS 套版开启时，展示当前选中文件夹的第一张成品图。
   - 未开启 PS 时，展示当前选中的最终图片。
   - 标题区域显示货号、模板批次或阶段来源。

2. 右侧为文件夹或图片陈列。
   - PS 套版开启时，展示套版后的文件夹卡片。
   - 未开启 PS 时，展示最终阶段图片卡片。
   - 新产物完成后立即追加，符合“生成一套显示一套”的流式体验。

3. 点开文件夹卡片。
   - PS 套版开启时，打开该 `模板批次 / 货号` 文件夹下所有裁剪后的成品图。
   - 卡片封面取该文件夹第一张图片。
   - 双击或打开操作沿用现有图片预览/系统打开路径能力。

## 多模板展示口径

多 PSD 模板同时套版时，文件夹卡片按真实输出结构展示：

```text
04-上架工作区/
├─ 模板A/
│  ├─ GZKJ-0001/
│  └─ GZKJ-0002/
└─ 模板B/
   ├─ GZKJ-0001/
   └─ GZKJ-0002/
```

展示卡片为：

```text
模板A / GZKJ-0001
模板A / GZKJ-0002
模板B / GZKJ-0001
模板B / GZKJ-0002
```

不按货号跨模板聚合，因为跨模板聚合会和硬盘真实目录不一致，用户点开后容易困惑。

## 页脚统计

页脚统计按实际开启步骤动态生成：

| 统计项 | 出现条件 | 口径 |
|---|---|---|
| 来源产物 | 始终出现 | 根据起点显示文生图、图生图、提取或已有印花数量 |
| 抠图 | 抠图开启 | 已完成抠图数量，失败数量可作为次要文本 |
| 侵权检测 | 检测开启 | 通过 / 疑似 / 拦截数量 |
| PS 套版 | PS 套版开启 | 文件夹数和成品图数 |
| 标题 | 标题生成开启 | 已生成标题数和失败数 |

未开启步骤不显示统计块，也不保留空位。

## 折叠区

中间产物默认收起，用户需要时可展开：

1. 印花产物区。
   - 文生图、图生图、提取、已有印花读取结果。
   - 文案随 `source.mode` 改变。

2. 抠图产物区。
   - 仅抠图开启时出现。
   - 默认收起。

3. 检测详情区。
   - 仅检测开启时出现。
   - 展示通过和未通过分组，保留风险值、风险等级和原因。
   - 默认收起。

4. 运行日志入口保持现有按钮，不放进主视觉区域。

## 数据结构影响

现有 `PipelineResultSection` 已能表达普通图片列表，但 PS 套版需要更明确的“文件夹组”展示。建议增量扩展共享类型，而不是另建一套结果系统。

建议新增可选结构：

```ts
interface PipelineResultGroup {
  id: string
  label: string
  subtitle?: string
  kind: 'folder' | 'image-set'
  cover_path?: string
  folder_path?: string
  template_batch?: string
  sku_code?: string
  items: PipelineResultImage[]
}
```

`PipelineResultSection` 可新增 `groups?: PipelineResultGroup[]`。旧的 `items` 保留，兼容现有图片列表。

PS 套版阶段写入：

- `section.key = 'print_products'`
- `section.groups` 按 `模板批次 / 货号` 聚合
- 每个 group 的 `cover_path` 取第一张输出图
- 每个 group 的 `items` 为该文件夹全部裁剪图

未开启 PS 时，主展示可继续从最后一个有效 `items` section 推导，不强制使用 groups。

## 前端组件拆分

建议在 `FullTaskPage.tsx` 内先局部拆组件，不做跨模块抽象：

1. `PipelineResultsPanel`
   - 负责选择主展示来源。
   - 负责布局主视觉、统计页脚和折叠区。

2. `PipelineHeroPreview`
   - 展示当前选中的 group 或 image。

3. `PipelineFinalResultGrid`
   - PS 开启时渲染 group 卡片。
   - PS 未开启时渲染图片卡片。

4. `PipelineStatsFooter`
   - 根据 config、progress.stats 和 steps 动态生成统计块。

5. `PipelineCollapsedSections`
   - 渲染中间产物和检测详情。

拆分只服务当前文件可读性，不引入全局 UI 框架。

## 验证标准

1. 最近完整任务区域不再渲染。
2. 开启 PS 套版时，主展示默认显示套版文件夹卡片。
3. 多模板套版时，卡片按 `模板批次 / 货号` 展示，不跨模板聚合。
4. 点开套版文件夹卡片能查看该文件夹全部裁剪图。
5. 未开启 PS 时，主展示显示最后一个开启阶段的产物。
6. 抠图未开启时，不显示抠图统计和抠图折叠区。
7. 检测未开启时，不显示检测统计和检测折叠区。
8. 文生图、图生图、提取、已有印花来源会显示不同来源文案。
9. 中间产物默认收起，用户可展开。
10. 流式运行时，新完成的一套结果会追加到主展示区域。
