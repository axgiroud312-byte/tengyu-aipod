# ADR-0012 — 完整任务最初版采用内置固定流程

## Status

Accepted

## Context

用户需要把“采集 / 生图 / 抠图 / 侵权检测 / PS 套版 / 标题生成”按顺序跑起来，
减少跨模块手动切换和重复选目录。

已有模块都具备独立 runner：

- 生图模块负责文生图、图生图、提取和抠图。
- 侵权检测模块负责批量检测和风险分类。
- PS 套版模块负责模板批次输出。
- 标题模块负责读取模板批次并写 `标题.xlsx`。

但通用编排引擎还没有完成。直接在 v1 做自由流程编辑器，会同时引入模板 DSL、
资源队列、暂停恢复、失败策略、断点续跑和上架串接，风险过大。

## Decision

v1 先做 **完整任务最初版**，采用内置固定流程：

```text
来源准备
  ├─ collection：从采集目录读取原图并提取
  ├─ txt2img：文生图产生印花
  ├─ img2img：图生图产生印花
  └─ existing_prints：已有印花直接进入后续流程
  ↓
可选抠图
  ↓
可选侵权检测
  ↓
PS 套版
  ↓
标题生成
```

首版边界：

- 不包含上架；上架仍由上架模块读取 `04-上架工作区` 后单独执行。
- 不做自由流程编辑器。
- 不做暂停 / 恢复 / 断点续跑。
- 只支持取消，且取消是尽力取消。
- 局部印花默认抠图，满印默认不抠图。
- 侵权检测可选；`block` 拦截，`pass` / `review` 放行。
- 因为 PS 套版是固定 step，完整任务首版只允许 Windows 启动。

实现上复用已有模块 runner，不模拟 UI 点击，不让服务端参与图片、LLM 或 API Key。

运行记录使用独立 SQLite 表：

- `pipeline_runs`
- `pipeline_steps`

这些表是固定完整任务首版的运行记录，不代表 v1.5 通用任务模型已经落地。

## Consequences

### Positive

- 快速满足最常用链路，不需要等通用编排引擎。
- 复用现有模块能力，避免复制业务逻辑。
- 上架仍保持独立模块边界，不影响 ADR-0004 的上架四层结构。
- 后续可以从真实使用反馈决定 v1.5 通用编排模板，而不是提前设计过度。

### Negative

- 首版不能自由增删步骤。
- 首版不能在 macOS 启动，因为固定包含 PS 套版。
- 首版失败策略固定，不能配置 `skip` / `pause`。
- 首版运行记录和未来通用任务模型存在一次迁移/兼容成本。

## Follow-ups

- v1.5 再设计通用编排引擎：多模板、资源队列、暂停恢复、失败策略、上架串接。
- 完整任务真实验收后，再决定是否增加“不含 PS”的 macOS 可运行模板。
- 如果用户强烈需要上架串接，先评估上架 profile 锁、店铺选择和失败恢复，再纳入通用编排。
