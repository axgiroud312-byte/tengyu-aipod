# ADR-0013 — 完整任务后续步骤采用显式开关

## Status

Accepted

## Context

ADR-0012 的完整任务最初版固定执行到 PS 套版和标题生成。真实使用中，用户经常只想先跑前半段：

- 采集目录 → 提取出印花
- 文生图 / 图生图 → 得到印花
- 可选抠图或检测后先人工查看

如果仍强制配置 PSD、标题和 Windows Photoshop 环境，会让前置能力无法单独运行，也会让晨羽云机在不需要时成为启动阻塞。

## Decision

完整任务仍保持固定顺序，不升级为自由编排器；但后续步骤改为显式开关：

```text
来源准备（必选）
  ├─ collection：采集 + 提取
  ├─ txt2img：Grsai 文生图
  └─ img2img：Grsai 图生图
  ↓
可选抠图
  ↓
可选侵权检测
  ↓
可选 PS 套版
  ↓
可选标题生成（依赖 PS 套版）
```

规则：

- 来源步骤始终执行；`collection` 来源包含提取。
- 抠图、侵权检测、PS 套版、标题生成都由页面开关控制。
- 标题生成依赖 PS 套版；关闭 PS 时标题自动关闭。
- 印花货号只在启用 PS 套版时必填。
- macOS 只在启用 PS 套版时禁止启动；只跑前置步骤可以启动。
- 晨羽云机只在当前开启步骤确实需要时校验。
- 图生图完整任务入口使用上传参考图，不再要求选择参考图文件夹；上传图保存到 `.workbench/pipeline-runs/{runId}/references/`。

## Consequences

### Positive

- 用户可以“配置到哪里，运行到哪里”，先验证前置产物再决定是否继续套版和标题。
- 不需要 PS 或晨羽的场景不再被相关配置阻塞。
- 保持固定顺序和现有 runner 复用，不引入 v1.5 的通用编排引擎复杂度。

### Negative

- 完整任务不再等同于完整全链路，UI 和文档必须明确“后续步骤按开关执行”。
- 运行记录会出现更多 `skipped` step，需要在摘要里正确解释。
- 图生图上传参考图会增加 `.workbench` 下的运行附件目录，需要随运行记录一起管理。

## Update: complete task workbench UX

2026-06-05 追加：完整任务页采用“上方配置、下方结果”的工作台体验。

- 配置切换页面后在当前软件会话内保留，重启后不恢复草稿。
- 完整任务里的侵权检测可为本次运行覆盖模型和 Skill，但不反写单独侵权检测模块默认配置。
- 文生图、图生图、提取和抠图等印花产物按预计数量预留槽位，完成一张显示一张，未完成显示加载中，失败槽位隐藏。
- 侵权检测结果按本次通过要求分成通过和未通过。
- 完整任务运行时允许单独模块按资源边界并行；Photoshop 和同一晨羽实例继续互斥。
