# ADR-0018 - PS 套版 v1 提供基础路径 B

**状态**：已采纳
**日期**：2026-07-15

## 背景

路径 A 使用 `placedLayerReplaceContents`，仍是兼容旧模板的可靠默认值。部分 300dpi 链接智能对象模板需要进入智能对象内部置入印花并显式控制 `fill` / `fit`，否则替换结果的尺寸和对位不可用。

旧 PRD、Spec 和参考资料把全部路径 B 能力安排在 v1.5。该范围同时包含基础内部置入和嵌套 / 共享智能对象递归处理，粒度过大，不能满足当前链接模板的基础套版需求。

## 决策

v1 在 Windows Photoshop 路径中提供基础路径 B，并由用户显式选择：

- 路径 A `replaceContents` 继续作为默认值。
- 路径 B `editSmartObject` 使用 `placedLayerEditContents` 和 `Plc ` 进入内部置入印花。
- 路径 B 支持 `fill`（默认）和 `fit`。
- 当前公开 UI / IPC 使用整个智能对象内部画布作为目标 bounds；底层 replacement 可为专用调用传入内部图层路径或名称。
- 程序置入层固定命名为 `__TENGYU_ARTWORK__`，下一次置入前只删除同名程序层，不删除模板原始层。
- 独立 PS 套版与固定完整任务使用同一配置；不改变完整任务顺序、上架边界或工作区目录。

以下能力仍留在 v1.5：

- 嵌套智能对象递归处理。
- 共享链接源去重和单次更新优化。
- `center` 和更复杂的对齐 / 裁切策略。
- 面向模板的内部占位层配置 UI。

本 ADR 只取代旧文档中“路径 B 整体延后到 v1.5”的时间安排，不改变 ADR-0007 的 Windows-only 决策。

## 影响

### 正面

- 300dpi 链接智能对象模板可在 v1 使用基础内部置入。
- 默认仍走路径 A，不影响现有模板。
- 高级递归能力继续留在 v1.5，没有把完整路径 B 一次性提前。

### 负面

- JSX 生成器需要同时维护路径 A 和基础路径 B。
- 路径 B 仍需在真实 Photoshop 和代表性 PSD fixture 上做人工 / live 验证。
