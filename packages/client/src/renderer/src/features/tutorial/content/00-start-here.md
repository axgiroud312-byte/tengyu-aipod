# 开始前准备

这页先把所有模块共用的前提讲清楚。采集、生图、PS 套版都会围绕同一个工作区运行；如果工作区没有选好，后面的保存路径和批次目录都会对不上。

![工作区与模块入口概览](/tutorial/start-here.png)

## 第一次使用先检查

1. 进入「设置」。
2. 在「工作区」里选择一个本地根目录。
3. 保存后，系统会自动创建 4 个业务工作区。
4. 按你要用的模块补齐对应 API Key 或本地环境。

## 工作区目录怎么看

| 目录 | 用途 |
|---|---|
| `01-采集工作区` | Temu 采集回来的原图。 |
| `02-印花工作区` | 生图、图生图、提取后的印花。 |
| `03-检测工作区` | 侵权检测后的分类结果。 |
| `04-上架工作区` | PS 套版输出的成品图，也是后续上架读取的位置。 |

业务图片只放在这 4 个业务工作区里。`.workbench` 是系统黑盒目录，用来保存缓存、日志和数据库，不需要手动整理。

## API Key 和外部环境

| 模块 | 必要配置 |
|---|---|
| Temu 采集 | 比特浏览器地址、可登录 Temu 的浏览器环境。 |
| 生图 | 百炼 API Key 用于生成提示词；Grsai API Key 用于付费生图；ComfyUI 路径需要晨羽云机或本地工作流。 |
| PS 套版 | Windows 电脑、Photoshop 已安装并能正常启动。 |

## 日志怎么用

每个模块顶部都有「日志」按钮。遇到失败时，先打开日志看最近几条：

- 采集日志看当前页面、扫描数量、下载失败原因。
- 生图日志看提示词生成、任务提交、保存路径和模型返回。
- PS 套版日志看 Photoshop 状态、模板扫描、当前处理的印花和输出校验。

## 推荐顺序

1. Temu 采集：把商品图保存到 `01-采集工作区`。
2. 生图：从原图提取或重新生成印花，保存到 `02-印花工作区`。
3. PS 套版：用印花替换 PSD/PSB 智能对象，输出到 `04-上架工作区`。
