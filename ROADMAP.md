# 腾域 aipod — Roadmap

> 当前路线图以已落地程序为准。旧 Trellis 任务清单已清理，不再作为开发入口。

## 当前 v1 范围

| 模块 | 状态 | 当前运行逻辑 |
|---|---|---|
| 工程骨架 | ✅ 已完成 | monorepo / Electron 客户端 / Next.js 服务端 / shared 类型 |
| 首次设置 | ✅ 已完成 | 设置页选择工作区，API Key 存 OS keychain，本地保存 setup 状态 |
| 采集 | ✅ 已完成 | 比特浏览器 + CDP，输出到 `01-采集工作区/{平台-时间}/` |
| 生图 | ✅ 已完成 | Grsai / ComfyUI / 晨羽，输出到 `02-印花工作区/{能力}/{任务名}/` |
| 侵权检测 | ✅ 已完成 | 百炼视觉模型，输出到 `03-检测工作区/{任务名}/{通过\|复查\|失败}/` |
| PS 套版 | ✅ 已完成 | Windows Photoshop，输出到 `04-上架工作区/{模板批次}/{货号}/` |
| 标题 | ✅ 已完成 | 百炼 + 云端 Skill，写入 `04-上架工作区/{模板批次}/标题.xlsx` |
| 上架 | ✅ 已完成 | 店小秘 + 比特浏览器 profile，读取 `04-上架工作区` |
| 完整任务 | ✅ 最初版 | 填写印花货号，来源 → 可选抠图 → 可选检测 → 等待套版同名印花 → PS 套版 → 标题生成；不含上架 |
| 服务端 | ✅ 已完成 | Admin 登录、客户记录、Skill 管理、公告/版本、健康检查 |

## 当前边界

- 客户端不做本地授权码、硬件绑定、设备指纹、客户端授权拦截。
- 服务端不代理生图、不代理 LLM、不保存图片、不保存用户 API Key。
- 服务端只派发 Skill / 公告 / 版本；Provider、模型清单、ComfyUI Workflow 和 API Key 都在客户端本地管理。
- 工作区固定 4 个业务目录：`01-采集工作区`、`02-印花工作区`、`03-检测工作区`、`04-上架工作区`。
- 完整任务的印花货号决定 `02-印花工作区/等待套版/{runId}/` 图片名，也决定 PS 后 `04-上架工作区/{模板批次}/{货号}/` 文件夹名。

## 下一阶段候选

这些不是当前待办，只是后续可选方向；启动前需要重新写 spec/ADR：

| 方向 | 说明 |
|---|---|
| 多平台上架 | 扩展 Temu / Shein 以外平台 |
| 通用编排引擎 | 把当前固定完整任务升级为多模板、可暂停恢复、可配置失败策略的任务系统 |
| 自动更新 | electron-updater、版本灰度、下载签名 |
| 代码签名 | macOS / Windows 正式分发所需 |
| i18n | 面向非中文操作员的界面翻译 |
