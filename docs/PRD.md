## 1. 产品定位

**腾域 aipod** 是面向**中国跨境电商运营**的桌面工作台，把"采集产品图 → 生成印花 → 侵权检测 → PS 套版 → 标题生成 → 店小秘批量上架"全流程串成一站式工具。

**目标用户**：跨境电商运营（POD 印花品类为主）。月流水 1-50 万人民币的中小卖家。

**核心痛点**：
- 全流程分散在多个工具里，跨工具迁移文件耗时
- 提示词、检测规则、上架字段全靠人工配置，不能复用
- 没有持续更新的"内置专家经验"

**解决方案**：把全流程集成在一个桌面客户端，配合云端持续派发提示词（Skill），让用户始终用上"最新版业务经验"；模型、Workflow、API Key 和上架规则留在客户端本地配置。

## 2. 用户故事

### Story A：从 0 到上架（全链路）

> 跨境运营**老王**在 Temu 上看到一款热销 T 恤。他用腾域**采集**了商品页的 8 张产品图；切到**生图模块**调用本地导入的 ComfyUI 提取工作流，从其中 2 张图提取出 2 个印花；进**侵权检测**，2 个都判定低风险通过；选 3 个不同的 PSD 模板（白T 正面 / 黑T 背面 / 卫衣正面）做**套版**，得到 6 个货号文件夹；**标题模块**给 6 个货号生成英文标题；最后**上架**到他的 2 个 Temu 店铺，几分钟搞定 12 条 listing。

### Story B：手里有印花直接套版（半链路）

> 设计师**小李**自己画了 20 张印花。她跳过采集和生图，把印花放入腾域的**02-印花工作区**，**套版模块**选 5 个模板做笛卡尔积输出到**04-上架工作区**（100 个货号），跳过侵权检测，**标题模块**生成西班牙语标题（卖 Mercado Libre），**上架**完事。

### Story C：纯生图试验（不上架）

> 个人卖家**小张**还在摸索风格。他用**生图模块的付费 Grsai** 反复试不同提示词风格的圣诞印花，跑出 50 张候选，存在 `02-印花工作区/文生图` 下。这周不打算上架，攒一批等下周决定再走后续流程。

### Story D：后台配置（你视角）

> 你（管理员）在浏览器打开 admin 后台：
> 1. 查看首次登录但还未开通的客户**小赵**
> 2. 给客户**小赵**授权并设置到期日
> 3. 上线新版本 v1.4.0，编辑 changelog，发布
> 4. 修改"标题生成 - Temu - 英文"的 skill 提示词，所有客户端 30 分钟内自动拉到新版本
> 5. 监控错误日志：发现"店小秘 SELECTOR_NOT_FOUND"上报集中爆发，推一次上架机器人选择器更新

│  │ 采集    │  │ 生图    │  │ 检测    │  │ 套版    │         │

│  └────┬────┘  └────┬────┘  └────┬────┘  └────┬────┘         │
│       │            │            │            │              │
│       └─────┬──────┴──────┬─────┴─────┬──────┘              │
│             ▼             ▼           ▼                      │
│    ┌──────────────┐  ┌──────────┐  ┌──────────────┐         │
│    │ BitBrowser   │  │ Adapter  │  │ TempFile     │         │
│    │ + CDP 适配器 │  │ Registry │  │ Manager      │         │
│    └──────────────┘  └──────────┘  └──────────────┘         │
│                                                              │
│    .workbench/workbench.db (SQLite) — 血缘、任务、本地配置状态     │

└─────────────────────────────────────────────────────────────┘
       │                  │                      │
       │ 直连             │ 直连                │ 联网
       ▼                  ▼                      ▼
┌──────────────┐  ┌──────────────┐  ┌──────────────────────┐
│ 比特浏览器   │  │ 晨羽智云     │  │ 腾域云服务器          │
│ 127.0.0.1    │  │ Grsai        │  │ Next.js + Postgres   │
│ + 店小秘     │  │ 阿里云百炼   │  │ /api/* + /admin       │
└──────────────┘  └──────────────┘  └──────────────────────┘

**核心原则**：
1. 业务计算和用户数据**全在本机**
2. 云端只管理**客户账号授权、Skill 系统提示词、公告和版本**，不代理任何用户数据
3. 模块按"业务能力"组织（而非按 provider）
4. 跨模块共享的能力（比特浏览器、CDP、临时文件管理）放 adapter 层
5. 客户登录复用旧 PHP 统一登录体系，Next 不负责微信 OAuth 本身、不保存 PHP `secret`

│   ├─ 文生图/{任务名}/

│   ├─ 提取/{任务名}/
│   ├─ 抠图/{任务名}/
│   └─ 等待套版/{完整任务runId}/     ← 完整任务进入 PS 前的最终印花副本
│
├─ 03-检测工作区/                    ← 侵权检测域（物理复制）

│       └─ 高风险/

│   ├─ {模板批次1}/

│   │   │   ├─ 01.jpg
│   │   │   └─ 02.jpg
│   │   ├─ {货号2}/

│   ├─ {模板批次2}/
│   └─ ...
│
└─ .workbench/                       ← 黑盒，用户不动
    ├─ workbench.db                  ← SQLite
    ├─ cache/                         ← Skill 缓存
    ├─ local-workflows/               ← 用户本地导入的 ComfyUI Workflow
    ├─ logs/                          ← 结构化日志

**强约束**：
- 4 个业务工作区目录下**只放业务图片**（标题 xlsx 只允许在上架批次目录；当前优先 `标题.xlsx`，兼容旧 `titles.xlsx`），元数据、状态、血缘**全部进数据库**
- **04-上架工作区 是上架域**，PS 套版和标题模块写，上架模块读
- 套版输入来自 `02-印花工作区`、检测通过候选清单或用户手工选择的印花目录
- 数据库追踪所有 artifact 的来源、provider、模型、skill 版本

## 6. 货号与任务模型

### 货号粒度

**货号 = 一个上架 listing**（方案 R）。

| 输入 | 货号数 |
|---|---|
| 8 张采集图 | 0（采集图不是货号）|
| 2 个提取的印花 | 0（印花不是货号）|
| 印花 A 套白T 正面 = 1 个 listing | 1 |
| 印花 A 套黑T 背面 = 1 个 listing | 1 |
| 印花 B 套白T 正面 + 黑T 背面 = 2 个 listing | 2 |
| **合计** | **3** |

**模板批次** = `04-上架工作区/{模板名}/`，每个模板自成一个上架批次目录，对应一份标题 xlsx。

### 货号格式

- 允许字符：英文字母（大小写）、数字、`-`、`_`
- 禁用：Windows 非法字符、空格、中文、保留名（CON/PRN/AUX/...）
- 长度 1-60
- 大小写比较不敏感（避免文件夹冲突）

### 货号生成策略

| 场景 | 货号生成 | 例 |
|---|---|---|
| 套版模块（默认）| 输入印花文件名（清洗后）| `pri_001` → 货号 `pri_001` |
| 完整任务 | 顶部填写的印花货号作为印花名/货号前缀 | `GzG0023` → `等待套版/GzG0023.png` → 货号 `GzG0023` |
| 预留自动货号策略（当前完整任务未启用）| `{平台}-{日期}-{序号}` | `TEMU-20260523-0001` |
| 模块独立测试 | `TEST-{时间戳}` | `TEST-20260523-153045` |
| 本地批量导入印花 | `{用户前缀}-{序号}` | 用户填 `BATCH1-` → `BATCH1-0001` |

### 任务两层

- **完整任务**：跨多模块顺序执行。v1 已有固定流程首版；v1.5 再升级为通用编排引擎驱动。
- **轻量任务**：单模块独立运行，每次执行也建任务（v1）

任务状态：`未开始` → `进行中` → `完成` / `失败` / `已弃 (重做时旧任务标记)`
同一货号同一时刻**最多一个"进行中"任务**。

## 7. 模块详细需求

详细 spec 见 `spec/` 目录，下面只列每个模块的 v1 核心需求。

### 7.1 采集模块

**v1 内置 7 个平台**：Temu / Ozon / Shein / TikTok / Shopee / 1688 / Mercado Libre

**核心模式**：
- **图池扫描**：用户在比特浏览器打开搜索页/列表页/商品详情页，腾域扫描当前页图片索引，用户勾选后下载。
- **商品详情主图采集**：商品详情页只采集左侧主图/轮播图，按商品页文件夹展示和保存。
- **点击采集**：用户点击图片保存原图到 `01-采集工作区/{platform}-{timestamp}/`，商品页图片进入 `商品页/{商品分组}`。
- **滚动采集**：用户滚动列表页，按规则自动保存图片到当次采集任务目录。
- **运行期日志**：采集页提供命令行式日志弹窗，实时显示扫描、下载、点击采集和会话诊断。

**约束**：
- 同时刻 workbench 内最多 1 个采集会话
- 一个会话只绑定一个**浏览器 profile**（与上架互斥）
- 不绕过登录/验证码/风控（用户自己在比特浏览器登录）
- 图池下载先在扫描阶段标记“散图/商品页”归属，保存和预览都按该归属展示。

详见 [spec/02-collection.md](./spec/02-collection.md)。

### 7.2 生图模块（统一模块按入口分 5 Tab）

**5 个入口 + 2 条本地运行路径**：

| 能力 | 付费（grsai）| ComfyUI 免费（comfyui-chenyu）|
|---|---|---|
| 文生图 | ✅ | ✅（文生图工作流可选）|
| 图生图 | ✅（4 种模式：参考构图/参考风格/构图+风格/自己写；“生图时带参考图”默认关闭）| ✅（多个工作流可选）|
| 提取 | ✅（图生图 + 提取 Skill）| ✅（提取工作流 + 同一套提取 Skill）|
| 抠图 | ❌（不内置）| ✅ ComfyUI 抠图工作流 / "黑白图 + 转遮罩混合"路径 |
| 提取后抠图 | ❌ | ✅（提取工作流 → 抠图工作流，只保留最终抠图结果）|

**用户视角**：
- 文生图：进入统一文生图页，先生成/审稿提示词，再在右侧“生图路径”选择 Grsai 或 ComfyUI 工作流。默认 Grsai。
- 图生图：保持原来的实现方式切换，Grsai 图生图和 ComfyUI 图生图不合并。
- ComfyUI 文生图 / 图生图 / 提取 / 抠图 / 提取后抠图：右侧显示运行云机选择卡，开关机仍去设置页处理。
- 生图页顶部提供“日志”按钮，弹窗实时显示提示词生成、任务提交、模型调用进度、完成/失败和保存路径。

**关键设计**：
- Grsai 生图只支持 `gpt-image-2` / `gpt-image-2-vip`，只走 `/v1/api/generate`
- 百炼文本/视觉模型清单在客户端本地维护，API Key 存 OS keychain
- 云端 Skill 只保存业务系统提示词；文生图和图生图提示词生成要求百炼返回 `{ "prompts": [{ "index": 1, "prompt": "..." }] }`，客户端只取 `prompt` 字段；提取 Skill 由 Grsai 和 ComfyUI 共用，直接作为每张源图的提取 prompt；侵权检测要求返回 `{ "risk_score": 0-100, "reason": "..." }`
- ComfyUI Workflow 由用户在客户端本地导入，不再上传云端服务器
- **抠图无纯付费路径**（付费模型抠不好），只有 comfyui 工作流或混合路径
- 生图结果按任务落盘：`02-印花工作区/{能力目录}/{任务名}/{印花ID}.png`；默认任务名为“能力-时间”，前端可自定义。

**晨羽智云集成**：
- 设置页只展示连接状态，不展示余额；API Key 存 OS keychain，保存后自动检测可用性
- 创建入口固定服务杭州慎思 POD：用户选择 POD 版本和 GPU 后创建实例
- 实例管理列出当前 API Key 下全部实例，主操作只保留开机、关机、设为默认云机
- 默认云机是生图页运行云机选择的默认候选；实际任务发送到用户在生图页选择的运行中云机
- 生图页 ComfyUI 路径显示运行云机、实例 UUID、ComfyUI 地址和刷新按钮；不在该卡片提供开机/关机按钮
- 默认设定时关机（晨羽侧定时关机 API），软件崩了不漏关
- 客户端调 ComfyUI 原生 HTTP API（不走晨羽 workflow/run 接口）
- 工作流 JSON 从客户端设置页本地导入，按能力分类保存

**Grsai 集成**：
- 用户填 API Key
- 直接调 `/v1/api/generate`（Grsai 原生接口）
- 仅支持 `gpt-image-2` / `gpt-image-2-vip`
- 支持 base64 参考图和像素尺寸；VIP 额外校验 16 倍数、长宽比和总像素范围
- 三种 replyType：json（同步）/ stream（流式）/ async（异步轮询）
- 国内 / 全球节点自动选择

详见 [spec/03-generation.md](./spec/03-generation.md)。

### 7.3 侵权检测

**Provider**：阿里云百炼

**模型选择**（用户可选）：
- qwen3-vl-flash（默认，便宜：输入 ¥0.15/百万Token）
- qwen3-vl-plus（平衡）
- qwen-vl-max（旗舰）

**风险三档**：
- 0-39 → `03-检测工作区/{任务名}/无风险/`
- 40-69 → `03-检测工作区/{任务名}/疑似/`
- 70-100 → `03-检测工作区/{任务名}/高风险/`
- **阈值用户可调**

**输入预处理管线**：
- 透明底加白（强制）
- 压缩（可选，默认 1024px JPG q=85）
- 用 `sharp` 库，并发受配置档限制
- Worker Thread 跑不卡 UI
- 临时文件由 TempFileManager 管理

**Skill 输入变量**：模型、是否输出依据、关注重点（多选）

**下游**：用户可一键把通过图登记到本地套版候选清单，PS 套版模块从候选清单读取可套版印花。

详见 [spec/04-detection.md](./spec/04-detection.md)。

### 7.4 PS 套版（最复杂，Windows only）

**PSD 模板扫描**：按 hash 缓存，扫描结果包括智能对象/参考线/裁切区/嵌套关系。

**JSX 实现两条路径**：
- **路径 A（v1）**：直接 `placedLayerReplaceContents`，简单可靠（xKeNcHii 极简模式）
- **路径 B（v1.5）**：进入 SO 编辑 + flattenImage + Plc，处理嵌套/共享 SO（joonaspaakko 复杂模式）

**v1 检测嵌套 SO 时**：UI 提示"v1 不支持，效果可能不理想"。

**UI 输入**：
- 印花文件夹路径
- 模板（多选）
- 替换范围（自动/顶层/全部）
- 适配方式（fill/fit/center）
- 导出格式（JPG/PNG）
- 裁切模式（none/auto/guides）
- 跳过已完成、失败重试

**进度面板**：总分组、完成、失败、跳过、已验证输出。

详见 [spec/05-photoshop.md](./spec/05-photoshop.md)。

**用户在 UI 填**：

**输出**：
- 在批次目录写入标题 xlsx（默认 `标题.xlsx`；货号 | 标题，两列）

### 7.6 上架

**Provider**：店小秘（通过 Playwright + 比特浏览器 CDP）

## 8. 完整任务与编排

### 8.1 v1 完整任务最初版

v1 已实现一个内置固定流程，不是自由流程编辑器：

```
来源准备
  ├─ 采集目录 → 提取
  ├─ 文生图
  └─ 图生图
  ↓
可选抠图
  ↓
可选侵权检测（block 拦截，默认 pass/review 放行）
  ↓
准备等待套版印花（按印花货号命名）
  ↓
PS 套版
  ↓
标题生成
```

边界：

- 不包含上架；上架仍由上架模块读取 `04-上架工作区` 后单独执行。
- 因为包含 PS 套版，v1 只允许 Windows 启动，macOS 上提示不可用。
- 只支持取消，不支持暂停 / 恢复 / 断点续跑。
- 复用各模块 runner，不模拟 UI 点击，不让服务端接触图片、LLM 调用或 API Key。
- 完整任务页只暴露采集+提取、文生图、图生图三类来源；已有印花仅保留为底层兼容入口，不作为当前主入口。
- 用户必须填写 **印花货号**。完整任务在进入 PS 前把最终印花复制到 `02-印花工作区/等待套版/{runId}/`：
  单张印花用 `{印花货号}.{ext}`，多张印花用 `{印花货号}-01.{ext}`、`{印花货号}-02.{ext}`。PS 输出的货号文件夹与这些印花文件名保持一致。
- 侵权检测不在完整任务页展开整套检测设置；完整任务读取侵权检测模块已保存配置，本页只提供启用开关、通过要求、压缩图片和配置摘要。
- 检测默认 `pass` / `review` 放行，也可选择“仅无风险通过”让 `review` 拦截。
- 右侧展示来源图、提示词、运行产物和运行摘要，减少完整任务执行时的黑盒感。

### 8.2 v1.5 通用编排引擎

v1.5 再把固定完整任务升级为通用编排引擎：

- 多个内置流程模板，例如完整链路、从印花开始、只生图、只套版、套版加上架。
- 两种执行模式：自动连跑 / 逐步确认。
- 可配置失败传播策略：任一步失败停止 / 跳过失败项继续 / 暂停等待用户处理。
- 支持更细粒度暂停、恢复、任务队列和资源锁调度。

详见 [spec/01-orchestration.md](./spec/01-orchestration.md)。

## 9. 服务器端

**技术栈**：Next.js + Postgres + Prisma + shadcn/ui admin

**职责严格边界**：
- ✅ 客户账号授权管理
- ✅ 客户授权校验
- ✅ Skill 系统提示词管理和派发
- ✅ 公告 / 版本通知
- ✅ Admin 邮箱密码登录
- ❌ 不负责微信扫码 OAuth 本身（复用旧 PHP）
- ❌ 不代理生图 / LLM
- ❌ 不存图片 / API Key / 业务数据
- ❌ 不保存 PHP `secret`

**当前授权边界**：
- 客户端启动前必须完成客户登录和授权校验
- 客户登录支持微信扫码和手机号验证码，登录请求走旧 PHP
- Next 以 PHP `uid` 作为客户账号唯一标识，负责 `pending / active / disabled / expired` 判定
- 首次登录客户自动创建为 `pending`，管理员授权并设置到期日后才可进入 Workbench
- 客户端启动时强校验，运行中每 5 分钟复查一次
- 发现 PHP 返回 `nologin: 1` 或授权失效后，客户端清空本地登录态并回登录页
- Skill 同步必须在客户授权通过后启动

**Admin 后台核心页面**：
1. 客户账号授权（头像、昵称、手机号、授权状态、到期日、备注）
2. 管理员账号（创建管理员、编辑名称/角色、启用/禁用）
3. Skill 系统提示词（生图、提取、侵权检测固定槽位）
4. 公告 / 版本通知

**客户状态**：
- `pending`：待开通，不能使用 Workbench
- `active`：已授权且未到期，可以使用 Workbench
- `disabled`：已禁用，不能使用 Workbench
- `expired`：`active` 但到期日已过，不能使用 Workbench

**容量假设**：2 核 2G 服务器承载 100-500 同时在线客户。500 在线客户每 5 分钟复查一次，约 1.7 rps；超过 2000 同时在线后再评估升配、拆分 Postgres 或调整复查间隔。

**部署**：2 核 2G 服务器（阿里云 / 腾讯云）+ Postgres + Nginx / Caddy。支持两条生产路径：服务器拉源码后用 `docker-compose.server.yml` 构建，或直接用 `docker-compose.image.yml` 拉取预构建镜像。月成本 ¥30-50 起。

详见 [spec/08-server.md](./spec/08-server.md)。

## 10. 安全模型

**总原则**：完美安全不存在，但腾域有天然优势：
- 核心资产（Skill）在服务端，模型、Workflow 和 selectors 留在客户端本地
- 业务依赖持续更新
- 目标用户是商人，盗版动力弱
- 跨境圈小众，传播链有限

**v1 必做的基础防御**：
1. Admin 使用独立 JWT 和管理员账号登录
2. 客户登录复用旧 PHP `uid + secret`，Next 只做授权校验且不保存 `secret`
3. 用户 API Key 只存 OS keychain
4. 服务端不保存图片、API Key、任务记录和业务文件
5. Skill 云端版本化，客户端本地缓存
6. 上架选择器和关键策略可随版本更新
7. 打包阶段再做 JS 代码混淆（JavaScript Obfuscator）

**v1.5+ 增强**：bytenode 编译、多点交叉验证、代码完整性校验。

## 12. 客户登录与首次启动

**启动门禁**：
1. App 启动
2. 主进程读取本地 `uid + secret`
3. 没有登录态时显示客户登录页
4. 有登录态时调用 Next `POST /api/customer-auth/verify`
5. Next 内部调用旧 PHP `/user/user/info` 校验登录态
6. `pending / disabled / expired` 显示不可用提示
7. `active` 且未到期后进入首次设置引导或 Workbench

**首次设置引导**：
1. 选择工作区
2. 设置 API Keys（可跳过，后续模块面板补填）
3. 完成 → 教程链接

客户授权通过前，不启动 Skill 同步。

**客户登录门禁验收标准**：
1. 未登录客户启动客户端时只能看到登录页。
2. 微信扫码或手机号验证码登录成功后，未授权客户显示“账号待开通”。
3. 管理员可在后台看到 `pending` 客户，并且授权时必须填写到期日。
4. `active` 且未到期客户可以进入 Workbench。
5. `disabled` 或到期客户不能进入 Workbench。
6. 同一账号在 B 机器登录后，A 机器 5 分钟内发现失效并回登录页。
7. Admin 邮箱密码登录不受客户登录影响。
8. Skill 同步不会在客户授权前启动。
9. Next 数据库不保存 PHP `secret`。
10. 服务端不接触图片、API Key、任务数据。

## 13. 错误、日志、遥测

- **日志**：`.workbench/logs/`，结构化 JSON，30 天自动清理
- **采集运行期日志**：采集页弹窗内存保留最近 `1000` 条，不落盘；用于当场排查扫描/下载进度。
- **生图运行期日志**：生图页弹窗内存保留最近 `1000` 条，不落盘；用于当场排查提示词生成、生图提交、模型调用进度和保存路径。
- **崩溃日志**：单独目录，下次启动询问是否上报
- **错误上报**：默认开启 + 可关，只上报错误类型/堆栈/模块/版本（不上报用户数据/图片/Skill 内容）
- **服务端聚合**：admin 后台看错误分布，发现集中爆发推热修复

## 14. 并发与资源管理

**全局并发上限**：默认 3，范围 1-10，用户可调

**百炼提示词生成**：生图提示词按 `100 条/批` 拆分，并发 10 批；1000 条即 10 次调用。单批 10 分钟超时后重试。当前文本/视觉提示词模型统一默认 `qwen3.6-flash`。

**资源互斥**：
- 比特浏览器 profile：BrowserProfileLock（采集 vs 上架）
- PS 实例：同时刻 1 个套版任务
- 晨羽运行云机：同一台云机同一时刻只允许一个腾域任务占用；任务内的 ComfyUI 队列仍由 ComfyUI 处理

**任务队列**：主进程内 priority queue，优先级 = 用户手动 > 编排引擎自动 > 重试

**暂停恢复**：按模块粒度
- 采集：会话级 / 图池级（图池扫描结果在前端内存中，重启后重新扫描）
- 生图：单图级
- 检测：单图级
- 套版：任务组级
- 标题：单货号级
- 上架：单 listing 级
