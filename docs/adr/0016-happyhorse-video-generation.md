# ADR-0016 — HappyHorse 视频生成作为独立模块接入

**状态**：已采纳
**日期**：2026-06-30
**决策者**：腾域 aipod 项目

## 背景

阿里云百炼 HappyHorse 提供文生视频、图生视频、参考生视频和视频编辑能力。腾域当前 v1 生图模块只处理图片印花，产物进入 `02-印花工作区`，并可能继续走抠图、侵权检测、PS 套版、标题和上架。

HappyHorse 输出 MP4 视频，不是印花，也不能直接进入现有图片链路。把视频能力塞进生图模块会破坏现有边界：生图模块的产物语义、工作区纯图片规则、完整任务路径和上架域都会变得含混。

## 决策

采纳：新增独立 **视频生成模块**，首版只接入 HappyHorse 的图生视频和参考生视频。

首版范围：

- 左侧导航新增 `视频生成`，位置放在业务导航最后，即 `上架` 后面。
- 新增 `05-视频工作区/`，视频结果自动下载保存到本地。
- 图生视频：1 张本地 **首帧图** + 可选提示词。
- 参考生视频：1-9 张本地 **视频参考图** + 必填提示词。
- 一次只生成 1 个视频，不做批量队列。
- 生成成功后必须下载 `video_url`，不能只显示 24 小时临时链接。
- 输出路径：

```text
05-视频工作区/
├─ 图生视频/{任务名}/0001.mp4
└─ 参考生视频/{任务名}/0001.mp4
```

任务名由用户填写；未填写时使用时间戳。若目标目录已存在 `0001.mp4`，必须报明确错误，不覆盖旧文件。

## 参数与默认值

页面暴露最少参数：

| 参数 | 默认值 | 说明 |
|---|---|---|
| 模型版本 | `happyhorse-1.1` | 页面展示版本，提交时按能力映射真实模型名 |
| 清晰度 | `720P` | 可选 `720P` / `1080P` |
| 时长 | `5` 秒 | 固定选项：`3` / `5` / `8` / `10` / `15` |
| 水印 | 关闭 | 提交 `watermark: false` |
| 参考生视频比例 | `9:16` | 支持官方全部比例 |

真实模型映射：

| 页面能力 | `happyhorse-1.1` | `happyhorse-1.0` |
|---|---|---|
| 图生视频 | `happyhorse-1.1-i2v` | `happyhorse-1.0-i2v` |
| 参考生视频 | `happyhorse-1.1-r2v` | `happyhorse-1.0-r2v` |

参考生视频支持官方全部比例：

`16:9`、`9:16`、`3:4`、`4:3`、`4:5`、`5:4`、`1:1`、`9:21`、`21:9`。

首版不暴露 `seed`，因为它只提升复现性，且官方不保证完全一致。

## 接口边界

首版复用用户本地保存的阿里云百炼 API Key，不新增 `WorkspaceId` 配置。

调用中国内地 DashScope 原生旧域名：

```text
POST https://dashscope.aliyuncs.com/api/v1/services/aigc/video-generation/video-synthesis
GET  https://dashscope.aliyuncs.com/api/v1/tasks/{task_id}
```

原因：官方推荐新的业务空间专属域名，但同时说明现有域名仍可正常使用。首版先避免让用户理解和填写 `WorkspaceId`；如果后续遇到账号、稳定性或性能问题，再把 `WorkspaceId` 做成可选高级配置。

创建任务必须带：

```text
Authorization: Bearer sk-xxx
Content-Type: application/json
X-DashScope-Async: enable
```

轮询状态：

- `PENDING` / `RUNNING`：继续轮询。
- `SUCCEEDED`：下载 `video_url` 到 `05-视频工作区`。
- `FAILED`：展示 `code` / `message`。
- `CANCELED` / `UNKNOWN`：停止轮询并展示明确状态。

首版只做“停止查询”，不做云端取消。用户停止后，阿里云侧任务可能继续运行并计费，UI 必须说清楚。

## 日志

视频生成页必须提供与采集、生图、完整任务一致的运行期日志入口：

- 页面上有 `日志` 按钮。
- 点击后打开命令行式日志弹窗。
- 日志区域自己滚动，追加新行时自动滚动到最新。
- 最近最多保留 `1000` 条，应用重启后清空。
- 支持清空当前内存日志。
- 日志行至少包含时间、级别、阶段、消息和关键详情。

运行期日志记录：

- 本地图片校验开始 / 成功 / 失败。
- 创建 HappyHorse 任务。
- 返回 `task_id`。
- 每次轮询得到的 `task_status`。
- 开始下载 `video_url`。
- 下载完成和本地保存路径。
- 用户停止查询。
- 失败错误码和错误信息。

同时写诊断日志：

```text
.workbench/logs/diagnostics/video/{taskId}.jsonl
```

诊断日志必须脱敏：

- 不记录 API Key、authorization、token、secret、password。
- 不记录图片 base64 原文。
- 图片只记录 mime、字节数、sha256、宽高和本地文件名。
- provider 原始返回可以记录，但必须先走统一脱敏。

## 输入校验

首版在点击开始生成后、提交 API 前先做本地图片校验；校验失败时不提交 API：

图生视频首帧图：

- 格式：JPEG / JPG / PNG / WEBP。
- 大小：不超过 20MB。
- 尺寸：宽高都不小于 300px。
- 宽高比：1:2.5 到 2.5:1。

参考生视频参考图：

- 格式：JPEG / JPG / PNG / WEBP。
- 大小：单图不超过 20MB。
- 数量：1-9 张。
- 尺寸：短边不低于 400px。

## 非目标

首版不做：

- 文生视频。
- 视频编辑。
- 本地视频上传转公网 URL。
- OSS 临时素材仓库。
- 批量队列、任务历史、断点恢复。
- 云端取消任务。
- `WorkspaceId` 设置项。
- 接入完整任务。
- 把视频当印花、检测输入、PS 套版输入或上架输入。

## 候选方案

| 方案 | 结论 | 原因 |
|---|---|---|
| 把 HappyHorse 放进生图模块 | 放弃 | 生图模块产物是印花图片，视频会污染现有链路 |
| 首版同时做视频编辑 | 放弃 | 输入视频要求公网 URL；本地视频需要 OSS/临时上传能力，范围过大 |
| 首版要求用户填写 WorkspaceId | 放弃 | 旧域名仍可用；多一个字段会增加配置成本 |
| 独立视频生成模块，只做图生视频和参考生视频 | 采纳 | 能覆盖当前需求，边界清楚，改动最小 |

## 影响

正面：

- 视频能力有独立入口和独立产物目录，不污染印花链路。
- 用户只需已有百炼 API Key，不需要理解 WorkspaceId。
- 自动下载 MP4，避免 24 小时临时链接过期导致产物丢失。

负面：

- `05-视频工作区` 改变了原“4 个业务工作区”的口径，后续实现需要更新启动创建目录逻辑。
- 暂不做视频编辑；如果以后要做，需要先设计本地视频上传到 OSS 的临时素材仓库。
- 暂不做任务历史；应用关闭后页面不会恢复上一条视频任务。

## Follow-ups

- 新增视频生成 spec，定义 IPC、保存路径、状态事件、诊断日志和 UI 细节。
- 实现 `HappyHorseVideoAdapter`，复用本地百炼 API Key。
- 实现 `05-视频工作区` 创建和路径保护。
- 为图片校验、模型映射、保存路径冲突、运行期日志格式和轮询状态写最小测试。
