# 腾域 Workbench 前端任务中心化重构设计

日期：2026-07-15
状态：设计已确认，待实施计划

## 1. 背景

腾域 Workbench 已具备采集、生图、侵权检测、PS 套版、标题生成、上架、视频生成和固定完整任务能力，但当前前端仍以平铺模块和长表单为主。完整任务的高价值自动化过程位于配置区域之后，用户启动前需要面对大量不常变化的设置，运行中也难以同时兼顾成果查看和其他模块操作。

本设计以日常操作员为主要用户，同时兼顾老板或客户演示。已确认的使用假设是：

- 完整任务约占日常使用的 70% 以上。
- 每次任务主要变化的是来源、任务名称和印花货号。
- 稳定配置通常存在 2-5 套，需要命名保存和一键引用。
- 用户既会留在运行页查看成果，也会切换到其他模块继续工作。

## 2. 设计依据

参考 Zan Design 的设计价值观、产品原则、语句表达和场景策略：

- 有效：围绕工作流交付业务结果，不只陈列功能。
- 高效：简化高频流程，帮助用户快速理解和决策。
- 反馈：操作后清楚展示当前状态和结果。
- 可控：不替用户决策，保留开关、修改、停止和恢复空间。
- 一致：导航、组件、状态、文案和业务现实保持一致。
- 说人话：先说明事实，再给出解决方法，避免技术黑话和责备语气。

参考资料：

- <https://design.youzan.com/design/design-values.html>
- <https://design.youzan.com/product-principle/principle.html>
- <https://design.youzan.com/strategy/sentence-expression.html>
- <https://design.youzan.com/strategy/scenario-strategy.html>

这些原则只用于方法和质量标准，不复制有赞的品牌、组件外观或专有内容。

## 3. 目标与非目标

### 3.1 目标

1. 将完整任务变为 Workbench 默认主入口。
2. 让高频任务只填写本次变量，其余设置通过执行方案复用。
3. 让每个完整任务阶段可直接查看、配置、启用和关闭。
4. 在运行中突出最终成果，同时准确表达流式多阶段并行状态。
5. 用户切换模块后仍可通过全局任务坞查看运行和异常。
6. 统一七个单步模块的页面语法、状态反馈和视觉系统。
7. 保留现有业务规则、路由、IPC 和 ADR-0015 固定流式流程边界。

### 3.2 非目标

1. 不实现 v1.5 通用可配置编排引擎。
2. 不将上架接入当前完整任务。
3. 不创建新的通用任务数据库或历史任务模型。
4. 不合并单步模块页面或改变模块业务职责。
5. 不修改服务端、图片存储、API Key 边界或工作区目录规则。
6. 不实现全局暗色主题；深色只用于图片成果画布。

## 4. 信息架构

```text
腾域 Workbench
├─ 生产
│  ├─ 完整任务（默认入口）
│  └─ 运行记录（仅固定完整任务）
├─ 单步工具
│  ├─ 采集
│  ├─ 生图
│  ├─ 侵权检测
│  ├─ PS 套版
│  ├─ 标题生成
│  ├─ 上架
│  └─ 视频生成
└─ 支持
   ├─ 教程
   └─ 设置
```

单步模块保持独立路由和一键直达。导航只做分组，不将多个模块合并为 Tab，也不使用搜索作为唯一入口。

### 4.1 启动逻辑

- 客户通过授权和首次设置后，默认进入 `/pipeline`。
- 存在运行中的完整任务时，进入该任务的运行详情。
- 同时存在多个运行中的完整任务时，默认打开最近更新的一项，并通过任务坞切换。
- 不存在运行中的完整任务时，进入快速启动台。
- 不新增营销式首页、数据大屏或模块宣传页。
- 不再使用采集页作为默认启动页。

## 5. 全局壳层

Workbench 使用三个稳定区域：

- 左侧导航：展开约 `188px`，折叠约 `56px`。
- 中央工作区：配置页限制阅读宽度，成果页使用完整可用宽度。
- 右侧任务坞：展开约 `310px`，折叠约 `44px`。

顶部栏约 `56px`，只显示页面名称、当前任务上下文和页面级操作。设置入口只保留在侧栏，避免重复。

任务坞在宽屏展开时推开内容；较窄窗口可覆盖中央工作区。折叠状态保存到本地，折叠后仍显示运行数量和异常标记。

## 6. 完整任务配置页

完整任务采用已确认的“方案 A：横向阶段总览”。

页面结构：

```text
执行方案选择 / 保存
→ 横向阶段卡片
→ 当前阶段编辑器
→ 本次执行摘要
→ 启动操作
```

### 6.1 阶段卡片

横向展示：任务起点、抠图、侵权检测、PS 套版、标题生成。

- 点击卡片：选择并打开该阶段设置。
- 点击卡片右上开关：直接启用或关闭阶段。
- 任务起点是必选阶段，不能关闭。
- 关闭阶段保留原配置，运行时记录为 `skipped`。
- 再次开启阶段时恢复原配置。
- 缺少配置时显示缺失数量，点击后定位缺失字段。
- 关闭 PS 套版时，标题生成自动关闭并锁定。
- 关闭 PS 套版前记住标题生成的开关状态；再次开启 PS 后恢复该状态，不擅自开启原本关闭的标题生成。
- 启动按钮不可用时显示首个可操作的阻断原因。

### 6.2 四类任务来源

任务起点先选择来源类型：

| 来源 | 业务含义 | 核心输入与设置 |
|---|---|---|
| 采集 + 提取 | 采集原图先提取为印花 | 采集任务文件夹、提取配置 |
| 文生图 | 根据文字要求生成印花 | 印花要求、生图路径、预计数量 |
| 图生图 | 根据参考图生成印花 | 参考图、参考方式、生图路径、预计数量 |
| 已有印花 | 从 `02-印花工作区` 中途开始 | 印花文件夹、开始阶段 |

已有印花支持从抠图、侵权检测或 PS 套版开始。开始阶段之前的步骤锁定为跳过，所选开始阶段保持开启；如需关闭该阶段，应改选更靠后的开始阶段。开始阶段之后的步骤仍可独立开关，并使用上一阶段的最新产物。

切换开始阶段时按以下规则更新阶段卡片：

- 从抠图开始：抠图开启，后续步骤按各自开关执行。
- 从侵权检测开始：抠图锁定为跳过，侵权检测开启，后续步骤按各自开关执行。
- 从 PS 套版开始：抠图和侵权检测锁定为跳过，PS 套版开启，标题生成仍可关闭。

采集得到的是原图，不在 UI 中称为印花；只有提取后才产生印花。

### 6.3 本次任务变量

以下字段属于本次任务，不写入执行方案：

- 来源文件夹。
- 参考图片和上传内容。
- 印花要求等本次内容输入。
- 任务名称。
- 印花货号。
- 分隔符。
- 印花类型。

同一软件会话内，这些字段仍通过草稿状态保留。不同来源类型分别保留自己的会话草稿；切换来源或执行方案后再切回时，恢复该来源之前填写的本次变量。

## 7. 执行方案

“执行方案”是 UI 正式术语，避免与 PSD 模板、流程模板和店小秘草稿模板混淆。

执行方案最多保存 5 套命名配置，不要求预先创建最低数量。默认引用上次使用的一套，并支持新建、覆盖保存、重命名和删除；删除当前引用方案前需要确认，删除后保留当前页面草稿，不自动套用其他方案。

### 7.1 保存内容

- 任务来源类型。
- 抠图、检测、套版、标题阶段开关。
- Provider、模型、Skill、工作流和并发设置。
- 生图尺寸、数量、参考方式和其他不包含图片内容的生成参数。
- 检测通过要求和图像预处理设置。
- 晨羽运行云机默认选择。
- PSD 模板路径、替换范围、裁切和导出设置。
- 标题平台、语言、模型、关键词和已有标题策略。

执行方案不保存 API Key、图片内容、任务记录或 §6.3 的本次任务变量。表单展示在任务起点不代表字段一定属于本次变量，保存时以本节白名单和 §6.3 排除清单为准。

### 7.2 存储与校验

首版使用 renderer `localStorage`，使用独立版本化 key：

```text
tengyu.pipeline.execution-plans.v1
tengyu.pipeline.last-execution-plan-id
```

存储内容通过 zod 校验，并带 `schema_version = 1`。损坏、字段不兼容或引用资源失效时：

- 不静默替换 Provider、模型、Skill、工作流、云机或 PSD。
- 对应阶段标记为“待配置”。
- 显示具体失效项和重新选择入口。

引用执行方案会更新来源类型、阶段开关和稳定参数，但不会覆盖当前来源下已填写的本次任务变量。

## 8. 运行详情

启动后仍位于完整任务路由，页面从配置态切换为运行态。已经启动的配置只读展示。

### 8.1 阶段导轨

- 遵守 ADR-0015，允许多个阶段同时显示为运行中。
- 每个阶段显示完成数、总数、失败数、拦截数和实际耗时。
- 不显示不可靠的剩余时间。
- 完成后切换为运行摘要，展示各阶段产出和耗时。

### 8.2 成果展示

- 深色成果画布展示最新完成的最终产物。
- 右侧成果墙流式追加新结果。
- 点击缩略图切换主预览。
- 开启 PS 时展示套版成品。
- 未开启 PS 时展示最后一个实际产生可继续产物的阶段。
- 中间产物默认收起。
- 页脚统计只展示本次启用的阶段。

完成后提供“按此方案再建任务”，复用执行方案并清空本次任务变量。

## 9. 异常、日志与停止

- 单张印花或单个货号失败只进入异常项，不打断其他项目。
- 错误信息使用“发生了什么 + 如何处理”，并包含阶段、路径、模型或资源上下文。
- 停止任务使用软停语义：不再接新项目，已经运行的单张允许完成。
- `interrupted` 或 `failed` 的完整任务保留现有手动续跑入口，不新增自动续跑。
- 首屏只显示最近三条关键记录。
- 完整日志仍通过独立弹窗查看。
- 检测全部拦截、零货号产出仍是正常完成并带警告，不标记为系统失败。

## 10. 全局任务坞

任务坞聚合当前会话中的完整任务和轻量任务：

- 显示运行、等待资源、完成、失败和软停状态。
- 等待资源时显示具体原因，例如店铺环境被采集占用。
- 点击任务返回对应模块或运行详情。
- 折叠后保留运行数量和异常数量。
- 展开/折叠状态写入本地 UI 设置。

首版使用前端 Zustand store 聚合现有模块事件，不新增通用 IPC、任务数据库或 v1.5 任务模型。完整任务历史仍来自现有 `pipeline_runs`；其他模块历史沿用各模块现有数据能力。

## 11. 单步模块页面语法

所有单步模块遵循：

```text
页面上下文
→ 输入与本次参数
→ 启动操作
→ 运行反馈
→ 结果与异常
→ 高级设置 / 日志
```

模块适配：

- 采集：浏览器环境和页面操作收紧为工具区，图池成为主体。
- 生图：保留五个能力入口，服务商和工作流属于当前能力设置，结果持续显示新印花。
- 侵权检测：输入和规则在左，按无风险、疑似、高风险展示结果。
- PS 套版：输入和 PSD 设置在左，按批次和货号展示成品。
- 标题生成：配置区与货号标题表并列，结果逐行更新。
- 上架：使用高密度状态表，突出店铺环境、货号状态、失败原因和重试操作。
- 视频生成：输入图片和参数在左，任务状态和 MP4 结果在右，不与印花流程混合。

上架模块内部仍严格保持 selectors、page-parser、action-executor、workflow 四层结构。

## 12. 设置、登录与首次设置

设置页分为：

1. 常规：工作区、比特浏览器、日志。
2. 模型与工作流：本地模型、API Key 绑定、Skill 同步、本地 ComfyUI 工作流。
3. 晨羽智云：连接、实例创建、开关机、默认云机和高级设置。

客户登录、授权和首次设置的业务门禁保持不变，只统一视觉和专业中性文案。

## 13. 视觉系统

采用“明暗双域”：

- 导航、表单、设置和表格使用明亮中性色。
- 图片预览、成果墙和放大检查使用深色画布。
- 不实现全局暗色表单主题。
- 删除渐变光效、模块插画横幅、发光阴影和装饰性英文角标。

颜色规则：

- 蓝色：主操作、当前选择、运行阶段。
- 绿色：成功、已就绪。
- 琥珀色：需要注意、等待资源。
- 红色：失败、危险操作。
- 灰色：关闭、跳过、不可用。

状态必须同时提供文字或图标，不能只依赖颜色。

组件尺度：

- 顶部栏约 `56px`。
- 导航项和常规按钮约 `36-40px`。
- 输入控件约 `38-40px`。
- 圆角以 `5-6px` 为主，弹窗不超过 `8px`。
- Noto Sans SC 作为主要 UI 字体。
- Lucide 图标统一为 `16-18px`。
- 动效控制在 `100-180ms`，并支持减少动态效果。

## 14. 文案与无障碍

文案使用专业中性语气：

- 先说明事实，再给解决方法。
- 不责备用户。
- 不直接暴露技术黑话。
- 避免“您”“亲”“绝对”“一定”等表达。
- 成功提示简短，错误提示包含阶段和必要上下文。

示例：

```text
套版失败，请确认 Photoshop 已启动后重试。
标题表正在被 Excel 占用，请关闭文件后继续。
店铺环境正在被采集占用，请结束采集任务后重试上架。
```

所有开关、Tab、阶段卡片、任务项和结果缩略图支持键盘操作，焦点样式清晰。正文、状态文字和深色画布满足可读对比度。

### 15.1 数据流

```text
用户编辑
→ PipelineDraftStore
→ 纯函数校验
→ 构建现有 PipelineRunConfig
→ window.api.pipeline.start
→ 现有 progress / completed 事件
→ 进度视图模型
→ 阶段导轨 + 运行详情 + 任务坞
```

配置与进度纯函数不依赖 React 或 IPC，输入输出可序列化并可独立测试。

## 16. 测试策略

### 16.1 单元测试

- 四类来源配置构建。
- 来源切换与分来源会话草稿。
- 阶段依赖和开关恢复。
- 标题依赖 PS 套版。
- 执行方案保存字段和本次变量排除。
- 执行方案版本、损坏数据和失效引用。
- 流式多阶段进度映射、拦截、失败、软停和运行摘要。

- 阶段卡片选择。

- 已有印花开始阶段与阶段开关联动。

### 16.3 E2E

- 四类任务来源启动路径。

### 16.4 工程与视觉验证

- 文本截断、按钮宽度、深色画布和焦点状态检查。

## 17. 实施分批

### 第一批：全局壳层和视觉令牌

- 新信息架构。
- 侧栏、顶部栏和页面容器。
- 明暗双域视觉令牌。
- 模块路由保持兼容。
- 首次设置“服务连接”步骤的专业中性文案和操作收敛。

### 第二批：完整任务配置和执行方案

- PipelineDraftStore。
- 横向可交互阶段卡片。

## 18. 验收标准

1. 启动 Workbench 默认进入完整任务快速启动台。
2. 四类来源可明确选择，并展示各自必要字段。
3. 五个阶段可点击设置，四个可选阶段可直接开关。
4. 关闭 PS 套版时标题自动关闭并锁定。
5. 执行方案可命名保存、一键引用，且不保存或覆盖本次任务变量。
6. 运行态准确表达多个阶段同时运行。
7. 最新最终产物和成果墙位于运行首屏。
8. 切换到其他模块后，任务坞仍显示运行和异常。
9. 单步模块保持独立路由，并使用一致页面语法。
10. 不改变完整任务顺序、上架边界、文件系统规则或 API Key 安全边界。
11. 相关单元测试、组件测试、E2E、type-check 和 build 通过。
